front-end 는 자바를 이용해서 하는게 아니고

java script 를 이용해서 하는것

Back-end 쪽에 있는 Servlet 과 JSP 가 있다.

이 두가지만 알았다고 해서 웹을 만들수 있는 것은 아니고

**web 을 개발하기 위해 사용될 수 있는 API**가 

- 기본적으로 jdk라는 것을 사용할 수 있고
- 그 다음에 데이터베이스(DBMS)를 이용하기 위한 JDBC 부분이 있다.

자바를 해본적이 없다 라고 하면 자바를 먼저 배우고

JDBC ,  DBMS 를  배운적이 없다 라고 하면 먼저 JDBC ,  DBMS 를 먼저 공부하고 오는게

**JDK . Servlet , JSP ,JDBC  이 네가지를 배워서 웹개발**을 하는데 

문제가 없을듯

추가적으로 배울것은 프론트엔드의  java script를 이용하는

html css 정도는 조금은 알고오는 것이 바람직 하다고 봄

이번 과정에서 목표는 **Java web을 개발하는 부분**

자바 웹을 개발하기 위해서는 

서블릿이라는 API 만 얹으면 된다.

그러나 **서블릿 만으로는 웹을 개발하기 어렵다.**

**문서를 출력하는 부분에 있어서 비효율적인 부분**이 있게된다.

(HTML 코드 출력문제)

그래서 문서에 대한 출력을 간결하게 만들어 줄 수 있는 

JSP 라는 도구를 이용하게 된다.

그런데 JSP 는 코드가 스파게티 처럼 만들어질 수 있다.

그래서 코드를 정리하게 되는 일이 발생하게 된다.

코드 정리를 잘하게 되면  JSP MVC 방식으로 만들게 된다.

그 이후에 Spring MVC 를 배우면 좋다.




# 2강 웹 서버 프로그램

### front-end , back-end  탄생



![Untitled](https://user-images.githubusercontent.com/89206108/162003945-8a7cf25e-27c5-4227-8d66-629bf3c15fc4.png)



일반적인 업무용 프로그램은

사용자가 어떤 폼에 입력을 하고 거기에 적절하게 요청을 하게 된다.

프로그램은 사용자 인터페이스에서 옮겨받은 그 주문을 받아서

적절하게 DB 쿼리를 한 다음에

그 결과물을 그 문서로 보여준다.

위가 업무용 데이터베이스 프로그램! 이었다.

그런데 시대가 지나며 아래와 같이 변했다.


![Untitled 1](https://user-images.githubusercontent.com/89206108/162003983-85c74d65-b447-4dc7-ae05-2dfd0298f590.png)



생산지에서 데이터가 만들어지면 원격에 있는 본사쪽에서

생산지에 있는 정보를 조회하고 싶은것

데이터라는 것을 맨날 가지고 가서 클라이언트 프로그램에 심어놓고 싱크하지 않는이상

실시간으로 보고싶다면 생산지에서는 데이터가 기록된 내용들을

실시간으로 전달하기 위해서 네트워크를 이용할것

요즘에 원격에 있는 네트워크라 함은 인터넷 이 될것이다.

**클라이언트 프로그램 > 요청자**

**서버 > 제공자**

한쪽에서는 클라이언트 라는 의미를 가진 프로그램과

한쪽에서는 서버 라는 의미를 가진 프로그램 으로 나눠지게 된다.

**과거**에는 프로그램을 만들때 그것을 구현하기 위한 언어가 다양했다.

그것을 만들때는 **클라이언트 프로그램과 서버프로그램이 동일한 언어로 만들어졌다.**

그러나 불편함이 생기기 시작

**두개가 동기화가 되어야 하는상황이라면**,

서버가 업데이트가 되어야 하는데 **서버쪽만 업데이트가 될 순 없는것**

고객정보가 달라졌으면 **클라이언트 쪽도 달라지면 등록하는것**도 달라져야 한다.

서버를 변경하는 건 문제가 없으나

클라이언트가 5명이라면 서버는 1번만 업데이트 하면된다.

그러나 클라이언트에게 전달했던 프로그램이 5명인 상황이 있다고 하면

5명의 상황을 다 다시 업데이트 해야되는것

여기에 대한 불편함이 큼.

**소프트웨어 업데이트가 과거에는 자동 업데이트가 아니라**

**설치와 재설치를 반복하는 과정**이었고

설치와 재설치를 반복하는 과정에서  다른 프로그램에 영향을 주는 문제도 있었다.

그래서 **설치와 재설치에 대한 부담이 커지고**

클라이언트에 배포에 대한 부담도 커지고

이런 부담을 줄일 수 있는 방법을 찾다보니 

**웹 이라는 것을 이용**하게 되는것.



![Untitled 2](https://user-images.githubusercontent.com/89206108/162004090-39361795-3076-43ee-bb2c-0429f98ac49e.png)


클라이언트 프로그램이  서버에 대한 데이터를 요청할 경우

소켓이라는 것을 이용해서 사용하는 것이 기본이었는데

소켓을 맨땅에서 사용하고자 하니까 너무 부담됨

그래서 직접 데이터를 포맷팅하고 전달하고 거기에 대한 응답을 받기 위한 것을

유효성검사 등을 하고 등등 여러가지 방법이 생기다가

원활하게 할 수 있는 방법을 찾다가 또 웹이라는 것을 생각하게 됨

재설치나 데이터 전송의 안전성 등을 해결하기 위해서

웹은 브라우저라고 하는게 있다.

요청에 있어서 응답하는 서버쪽이 제공하는데 있어서 

요청에 대한 응답을 할수가 있는데

요청을 하면 거기에 대한 응답을 찾아서  보내주게 된다.

여기에 웹 프로그램이라는 것을 이용해서

우리가 생각하고 있는 서버와 클라이언트 프로그램을 만들수가 있다.

웹 프로그램이라고 하는것은 

곧, 서버 클라이언트 프로그램이 아니라

**서버클라이언트 프로그램을 웹 프로그램에 얹을 수 있다 는 것을 얘기하는것**


![Untitled 3](https://user-images.githubusercontent.com/89206108/162004148-d8b25fc4-2de5-4e8d-bc7f-68c0252d9733.png)



웹은 기본적으로 페이지를 요청하고 페이지를 찾아서 페이지를 제공하는

그런 방식이었는데

이 페이지라는 것을 잘 생각해보면

생산내역, 상품목록을 보여 달라 라는것을 웹페이지로 만들어 보여줄수 있음

문제는 서버쪽에서 데이터베이스 이용해서 만들어 주는 작업이 필요함

그런것들을 **단순히 페이지를 찾아서 전달해주는 것이 아닌,**

**디비연동해서 동적으로 만들수 있는 환경을 추가하게 된다.**

그래서 서버프로그램이 간단히 얹어질 수 있는 환경을 만들게 되고

클라이언트 쪽에서는 사용자 요청이라던지 ,

서버가 전달하는 목록을 보여주는 것을 브라우저를 이용하게 되는것.

**서버쪽에 문서를 요청하는 데 문서가 동적으로 만들어져서**

사용자 목록을 만들어준다.

그러면 **클라이언트 프로그램을 특별히 만들어줄 필요가 없어짐**

왜냐면 문서로 전달 되기 때문에 브라우져만 있으면 되는거고,

클라이언트 프로그램을 재설치 할 필요가 없어진다.

이런 방식을 서버프로그램과 클라이언트 프로그램방식으로  나눠지게 되면서

웹개발을 이용한 서버 프로그램을 만들게 되는것.

과거에는 클라이언트 프로그램+ 서버프로그램 줄여서

CS 프로그램 이라고 했는데

웹으로 오면서 클라이언트 프로그램이 사라지게 되는것

과거에는 웹 개발자라고 하면 서버프로그래머 라고 했지만

요즘에는 **자바스크립트**라는 것도 생김

그래서 페이지 요청이 아닌,

**데이터 요청으로 바뀌게 됨.**

그래서 브라우져 단에다가 웹기반으로 클라이언트 프로그램을 만들게 됨.

그래서 **클라이언트 쪽을 요즘은 웹의 프론트단을 개발하는 사람.**

**웹의 서버단을 개발하는 사람**

이렇게 해서 프론트엔드,  백엔드 라는 개념이 탄생하게 된다.

우리는 백엔드 쪽을 배우게 된다.

그럼 우리는 백엔드쪽을 만들기 위해  추가적인 실행 환경이 필요하다.

3강에서는 서버쪽 구성을 살펴볼 것.

톰캣을 실행할 수 있는 환경이 아닌것

 jdk환경변수가  JAVA_HOME 이 환경변수 등록이 안되서 그런 것일 수 있음

JAVA_HOME 이란

JDK 를 필요로 하는 어플리케이션이 있을것 

대표적인게 JAVA_HOME 이 있는데 그게 바로 톰캣.

톰캣은 이 디렉토리가 즉,  jdk 가 어디에 설치되어있는지 알아야 하는데

그건 사용자마다 설치한 곳이 다르기 때문에 

그 jdk가 설치되어있는 곳을 약속해놓은게 

JAVA_HOME

자바홈에 등록이 안되어잇으면 jdk 를 확인 할수 없기 때문에 

환경변수 설정을 해주는것

그래서 톰캣 사용전에 jdk 환경변수 설정이 필수 인것.

아니면 다른 톰캣이 동작 중이어서 포트가 사용중이어서 스타트업 배트 파일이 실행이 안되는 것일수도 있음
